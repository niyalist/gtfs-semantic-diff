# SD4 検証ログ: 運行日カレンダービュー (2026-07-23)

設計: docs/design/service_days.md §3。実装:
- bundle `_calendar_view` (feed_overview.calendar_view.old/new): 日付ごとに
  「その日実際に走る世界」を実効運行日集合 (SD1/SD2 と同一定義) から決定。
  sym = 走る週次型ラベル (曜日集合がその日を含むもの優先・広い区分優先)、
  swap = 週次型は走るが曜日が合わない (振替)、special = 特定日 service 運行、
  period = calendar 端点で区切った期間番号。窓上限
  `[report] calendar_view_max_days` (550) 超は省略 (国家規模)
- viewer `CalendarView.svelte`: 月×日の年間グリッド新旧2枚 + 凡例。
  **記号第1チャネル** (色弱原則): セル = ▲?+1文字+◆?
  (平/土/休/週/毎/曜/「・」=運行なし)。期間の切れ目は太左罫線+交互背景
  (補強)、①②… の期間一覧をヘッダに。ツールチップに日付+説明

## 改訂 (2026-07-24): カレンダー描画の廃止 → 文字要約へ

表現の見直し壁打ちを経て、3案 (差分ハイライト/例外台帳/複合) × 4データセット
(桑名・佐賀・名古屋・PRT) のモック比較を実施。結論 (ユーザー決定):
**カレンダー描画は行わず、文字による「運行日の要点」のみを示す**。

- 根拠: カレンダーが伝える本質 (改正日・例外日・運休日・掲載範囲) は5行程度の
  文に圧縮でき、月グリッドは縦空間・i18n・セル情報容量の負担に見合わない。
  文字は本レポートの思想 (認知単位を言葉で語る) と一致する
- 実装: feed_overview.service_days_note — 掲載範囲/両方が重なる期間/
  運行内容が変わる日 (便内容署名の日付単位比較・ID 非依存)/曜日と異なる
  ダイヤで走る日/運行のない日。連続日はラン表示 (`note_runs_max`=8)
- 「任意の2時点で動く」前提: 期間が重ならないペア (PRT 実測) では日付照合を
  行わず「期間が重ならない2時点の比較」と明示する縮退を実測確認
- 実測: 桑名 = 変わる日 85日が「7/11〜10/3」の1ランで出る (改正日が文で創発)。
  佐賀 = 振替に 4/29・5/4〜5/6・7/20・8/11 (祝日がデータから浮かぶ)
- 第1部「特定日・運行日なしの内訳」の日付列挙も撤去 (日数+期間のみに戻す)。
  具体日付は路線ページ「特定日」タブのその場解説が受け持つ
- カレンダー実装 (CalendarView.svelte 等) は削除。試行の最終形は
  git 953176a に保存

## 改訂2 (2026-07-24): 「比較の概要」への統合 (データ/条件/結果の3段)

「同梱世代と比較範囲」と「運行日の要点」は重なる情報が多かったため、第1部の
1ブロック「比較の概要」に統合 (ユーザー指摘)。構成は3段:

1. **データ**: 旧/新それぞれの feed_version・feed_info 期間 (任意ファイル —
   なし/記載なしを明示)。feed_info の期間と実データの窓が食い違う場合は
   「(実データは C〜D)」を併記
2. **比較の条件**: 窓の位相で分岐 —
   - A1 重なる: 「両方が重なる期間: A〜B」。同梱世代 (Bb) があれば
     「同梱世代を検出 — 比較は X〜Y の世代同士」+ 対象外世代の内訳
   - A2 重ならない: 「期間が重ならない2時点の比較 (日付単位の照合は行わない)」
   - A3 期間不明・長期: 「日付単位の照合は行っていない (時刻表全体の比較)」
3. **結果**: 変化の形で言い分ける (ユーザー指摘の語彙) —
   - C0 変化なし: 「重なる期間内に運行内容の変化はない」
   - C1 切替型 (終端まで): **「運行の切替日: ○月○日」** (期間でなく日付を主役に)
   - C2 有界型 (お盆等): 「A〜B は運行内容が異なる (期間限定の変化)」
   - C3 全面型 (重なり全日): 「全日で異なる (切替日はこの期間より前の可能性)」
   - C4 混合: 上記の組み合わせ + 計N日
   + 曜日と異なるダイヤで走る日・運行のない日 (各世代)

パターン別実測: 桑名 premerge→coex = A1×Bb×C1 (「運行の切替日: 2026-07-11」)、
佐賀 = A1×Bb×C3 (改正日が重なりの先頭 = 正直に「期間より前の可能性」)、
桑名ローリング隣接 = A1×Ba×C0、PRT = A2。JSON は feed_overview.data_briefs を
加算 (comparison_scope の old/new_feed_info は互換のため維持)。
