## 実装内容

通常モード (レポート) = プレゼンテーションモデルによる路線ページ4部構成:

1. **①路線概要**: 代表停車列 (中央を省略表示)、方向グループと系統一覧 (新旧便数・
   新/廃 表示)、地図は折りたたみ (系統別に**色+線種**で区別、凡例付き — 原則5)
2. **②変化のサマリー**: Lev.1 バナー (下位省略) / Lev.2 ◆系統の新設・廃止
   (吸収した Lev.3 を配下に) / Lev.3 ○経由の変更 (影響率「48便中30便」表示、
   クリックでパターン差分) / Lev.4 ▲▼増減便 (「純減5 (増0・減5)」) /
   Lev.5 ・微調整の一行集約
3. **③時間帯別本数**: 集計行 (太字) + 系統内訳行。セルは「旧→新▲/▼」の
   数値+記号が第1チャネル、色は補強のみ
4. **④新旧時刻表**: 出版時刻表形式 (1列1便・全停留所・停留所名は sticky)。
   **旧/新/差分の3切替**。差分表示: 時刻変更セル = 太字+旧時刻を取り消し線で併記、
   廃止便列 = 取り消し線+「廃」、新設便列 = 下線+「新」、変更列ヘッダ「＊」。
   凡例文を表示 (色に依存しない読み方の明示)

検証モード = W1 のイベント単位 UI 全体をそのまま退避 (全体サマリ・イベント羅列・
evidence → RawDiff 生値・路線に紐付かない変化 (運賃等)・explained_ratio・残差)。
ヘッダのトグルで切替。**削除はしていない** (R9・設計原則3)。

大規模改正対策 (八戸): 既定は Lev.1/Lev.2 を含むページのみ展開、他は折りたたみ。
「すべて展開 / すべて折りたたむ」操作を提供。時刻表は変更のあるテーブルが
4枚以下のときのみ既定展開。

## レビュー対応 (2026-07-06 第1ラウンド)

1. **時刻表の空欄の書き分け** (指摘対応): 運行区間外 = 純粋な空欄 /
   運行区間内で経由しない停留所 = 「‖」(縦の＝、セル中央)/ 経路上だが時刻なし (通過) = 「✓」。
   presentation モデルで「経路外 (null)」と「通過 ("")」を区別するよう拡張
2. **表示上変化のない時刻に差分マークが付く問題**: changed_positions を
   表示粒度 (分) で比較するよう修正 (秒だけの差は変更扱いしない)。
   分単位で変更が残らない列は「＊」マーク・変更列数からも除外
3. **永井運輸の同時刻重複便**: データ重複ではなく day_type 分類の問題と特定 —
   「年末・年始」サービス (calendar_dates 5日) が曜日多数決で weekday に分類され
   平日テーブルに同居していた。運行日数 ≤ short_service_max_days (10, config) の
   calendar_dates サービスは「特定日」に分類するルールを追加 (detection.md §0 同期)。
   修正後: 平日テーブルの同時刻重複 0、年末年始166列は特定日テーブルへ分離

## レビュー対応 (2026-07-06 第2ラウンド、豊前市の実例から)

4. **Lev.3 の第2段階マージ (一般ルール)**: 人が認識する変化の単位 =
   「(追加停留所集合, 削除停留所集合)」をキーに、方向・系統・変形種別
   (延伸/挿入/迂回) を跨いでマージ。系統ごとの内訳 (影響便数・パターン差分) は
   展開時に表示。素材は B群イベントでなく trip_delta.modified の trip 単位集計に変更
   (direction_id の無いフィードでの帰属精度向上)。
   豊前市: 岩屋線 15系統分の変化 → 1ユニット (追加2停・削除2停, 110便中66便)。
   轟線は追加/削除集合が異なる3変種 → 3ユニット (忠実な挙動)
5. **新版で停まらなくなった箇所**: ‖ でなく取り消し線付き旧時刻を表示。
   逆に新規に停まる箇所は下線 (新設寄りの表現)
6. **新旧どちらかにしか無い停留所**: 行名をグレー + 【廃】/【新】マーカー
   (色以外のチャネル併用)。存在しない世代のセルは ‖ でなくグレーの「・・」

## レビュー対応 (2026-07-06 第3ラウンド)

7. **廃止便列の背景を濃いグレーに** (廃止感の表現。取り消し線が第1チャネルのまま)
8. **行名グレーを表示モード連動に**: 旧表示では新設停留所をグレー (【廃】は黒)、
   新・差分表示では廃止停留所をグレー (【新】は黒)
9. **列幅の固定**: 1便 = 3.4em 固定・中央揃え (「12:34」5文字基準)。
   ‖ 等の記号がセル中央に揃う
10. **表示用ペアリング** (畑線・フィーダーバスの指摘): 会計上 removed+added の
    便でも、同一バケット (方向×曜日) 内で発時刻差 ≤ pair_max_shift_min (60分,
    config) の組を貪欲・決定的にペアリングし、1列 (retimed/rerouted) で差分表示。
    根本原因の実例: 豊前市は新世代で direction_id 付与 + trip_id 空白除去により
    厳密署名が組めなかった (時刻・停車列は完全一致)。修正後: 畑線 14便が
    全て実質無変化の1列表示、フィーダーバスは大半が rerouted ペアになり
    純減便 (2〜3本) のみが「廃」として残る。events / accounting は不変
