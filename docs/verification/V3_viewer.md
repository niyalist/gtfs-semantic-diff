## 実装内容

通常モード (レポート) = プレゼンテーションモデルによる路線ページ4部構成:

1. **①路線概要**: 代表停車列 (中央を省略表示)、方向グループと系統一覧 (新旧便数・
   新/廃 表示)、地図は折りたたみ (系統別に**色+線種**で区別、凡例付き — 原則5)
2. **②変化のサマリー**: Lev.1 バナー (下位省略) / Lev.2 ◆系統の新設・廃止
   (吸収した Lev.3 を配下に) / Lev.3 ○経由の変更 (影響率「48便中30便」表示、
   クリックでパターン差分) / Lev.4 ▲▼増減便 (「純減5 (増0・減5)」) /
   Lev.5 ・微調整の一行集約
3. **③時間帯別本数**: 集計行 (太字) + 系統内訳行。セルは「旧→新▲/▼」の
   数値+記号が第1チャネル、色は補強のみ
4. **④新旧時刻表**: 出版時刻表形式 (1列1便・全停留所・停留所名は sticky)。
   **旧/新/差分の3切替**。差分表示: 時刻変更セル = 太字+旧時刻を取り消し線で併記、
   廃止便列 = 取り消し線+「廃」、新設便列 = 下線+「新」、変更列ヘッダ「＊」。
   凡例文を表示 (色に依存しない読み方の明示)

検証モード = W1 のイベント単位 UI 全体をそのまま退避 (全体サマリ・イベント羅列・
evidence → RawDiff 生値・路線に紐付かない変化 (運賃等)・explained_ratio・残差)。
ヘッダのトグルで切替。**削除はしていない** (R9・設計原則3)。

大規模改正対策 (八戸): 既定は Lev.1/Lev.2 を含むページのみ展開、他は折りたたみ。
「すべて展開 / すべて折りたたむ」操作を提供。時刻表は変更のあるテーブルが
4枚以下のときのみ既定展開。

## レビュー対応 (2026-07-06 第6ラウンド: 地図の線)

17. **オフセットを「進行方向の右側レーン」方式に修正**: 前実装 (中心対称の係数) は
    line-offset が進行方向基準のため、往路と復路が地理的に同じ側へ寄って重なる
    欠陥があった。各 (方向グループ, leg) 内でレーン番号 0,1,2… を割り当て
    (双方向グループは1から)、全系統を進行方向の右側へオフセット — 対向方向は
    自動的に反対側に並ぶ (道路の車線と同じ定石)。MapLibre の line-offset が
    コーナー接合を含め幾何処理するため、ジオメトリ加工は不要。
    ネットワーク全体の交錯最少化は対象外 (要件通り)
18. **線幅のズーム連動を強化**: z10:3.5px → z13:6 → z15:9 → z17:14
    (廃止系統は一段細く)。オフセット量も同スケールで拡大 (レーン間隔が保たれる)

## レビュー対応 (2026-07-06 第8ラウンド: 線描画のデザイン刷新)

20. **点線の廃止 — 線は常に連続**。線種バリエーションは dash でなく線の
    **デザイン (構造)** で出す3種に変更:
    - solid: 単色線
    - stripe: 同幅の濃灰 (#333) ダッシュを色線の**上に重ねる** — 隙間は下の
      色線が埋めるため途切れず、2色交互の縞々になる
    - double: 細い白芯を重ねた二本線
    縞の第2色を白にしなかったのは、白縁取りと同化して「途切れ」に見えるため。
21. **色弱対応パレットの再設計**: Okabe-Ito 系を暗めに調整した濃色5色
    (#0072B2 / #D55E00 / #008A5E / #7B3294 / #B0004E)。5色 × 3デザイン = 15組
    が一意。色と線デザインの2チャネルで識別 (原則5: 色だけに頼らない)。
    廃止系統の不透明度も 0.6 → 0.75 に上げ淡色化を抑制。
22. **白縁取り (casing)**: 全系統の線の下に白線 (本体より片側 1.5〜3px 太、
    ズーム連動) を敷き、地図とのコントラストを確保。描画順は
    「全縁取り → 全色線 → 縞/白芯」の3段 — 縁取りが隣レーンの線を上書きしない。
23. 凡例も同一デザインで再現 (背景が白のため、縁取りの代わりに淡灰帯で白芯を可視化)。
