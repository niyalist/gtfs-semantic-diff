# V4 検証: 停留所の変化章 (レポートモード)

実施日: 2026-07-07。設計: docs/design/coverage_and_nonroute.md V4。
ユーザー指示: 路線に紐付けずレポートに章として追加 / 廃止・追加は1事象1行に
せずまとめる / 改称は重要・乗り場はマイナー / 地図併用 /
**ページができた後の微修正が重要 (レビューラウンドで育てる)**。

## 実装 (第1イテレーション)

- presentation に `stop_changes` を追加 (`_stop_changes`): D群イベントを
  停留所クラスタ単位に集約
  - **改称・移設**: 1停留所1項目 (座標・関係 route_group 付き)。改称は
    旧名取り消し線 → 新名太字で最上位に表示
  - **新設・廃止**: **関係 route_group の組ごとにまとめる** — 路線廃止で
    一斉に消える停留所は1項目になる。チップ表示 (新=下線枠 / 廃=取り消し線、
    app.css の .stop.added/.removed を再利用)
  - **乗り場 (PLATFORM_*)**: マイナー扱い — 折りたたみに種類別件数で集約
- ビューア: StopChangesPage (章、変化のある路線章の直後) + StopChangeMap
  (専用地図: 種別を【改称】【移設】【新設】【廃止】のラベル接頭辞で表す
  (第1チャネル)、色は補強。移設は旧→新を破線で結ぶ)
- 既定の展開: 改称・移設があるときのみ展開 (新設・廃止のみは路線側で説明
  されることが多いため折りたたみ)
- コア・会計・イベントは不変。テスト1件追加 (計150件)

## 追記 (2026-07-07): レポートの4部構成化

ユーザー提案によりレポートモードを4部構成に再編。停留所章は**第2部**となり、
変化がある場合は**地図を最初から開いた状態**で表示するよう変更 (章の折りたたみ
ラッパを廃止し、部見出し直下に要約行 → 地図 → 改称/移設/新設/廃止の順)。
第1部 (フィード全体: ファイル対応表・期間・曜日別便数・フィード級イベント) と
第4部 (その他 = 未収載イベントの種類別件数) を新設。路線章は 3.x 番号。
