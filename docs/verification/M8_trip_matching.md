# M8 検証: trip matching v2 (便対応付けの大域割当化)

実施日: 2026-07-08。設計: docs/design/trip_matching.md。
発端: docs/verification/trip_identity_survey.md (八戸の反例と根本原因)。

## 実装

- events/tripdelta.py: ①内容署名の完全一致 → ②残りを (対応family, day_type)
  ブロック内でコスト最小割当 (Δt_shared = 共有停留所時刻差の中央値 + 経路 LCS率
  + ID 一致は弱い事前)。受理ゲート 2つ + 重みは config [matching]
- 表示層の後付けペアリング (③ _pair_leftovers) を廃止し、④・Lev.3・Lev.5 は
  delta.modified を直接使用 (表示と検出の対応付けが一致)
- B群 patterns / C群 frequency を ID 跨ぎ対応に拡張 (旧 ID の row_removed +
  新 ID の row_added を evidence として消費 — これを怠ると残差になる。
  実装中に八戸 explained 0.9993→0.9881 の退行として検出し修正済み)

## 感度分析 (実装前)

クリーン正解 (ID 安定 + 時刻整合) 11フィード・正解対359 vs 非正解対3,791:
- 正解: Δt_shared p99 = 5分・LCS率 最小 0.82 / 非正解: Δt≤5分は 1.8%のみ
- コスト分布: 正解 p95 = 0.1 vs 非正解 p5 = 0.77 — **大きな二極化**で閾値に
  敏感でない。既定値 (min_route_sim 0.5 / max_shift 60分 / w_id 0.05) を確定

## 検証結果

1. **八戸の反例が解消** (ユーザー指摘の表を再現確認):
   旧11:30(32停)→新11:30(28停) の短縮振替が1列に組まれ (系統跨ぎ 600222-2→
   600212-2)、15:00→15:00・17:25→17:25・18:25→18:25 が正しく対応。
   「11:30→15:00」のようなオフバイワン誤対応は消滅
2. **合成テスト**: 連番 ID スライド+経路変更+短縮振替 (八戸型) を単体テスト化。
   既存の畑線型 (ID 全張替え)・シフト上限・Lev.3 回収テストもすべて通過 (158件)
3. **クリーン正解での recall = 0.992** (355/358)。未回収3件はすべて
   family/day_type がブロックを跨いだ既知の v1 制限 (内容同一・時刻同一)。
   ※初回計測 0.81 はクリーン判定が v2 出力を使う循環で八戸の偽正解が混入した
   誤り — 生の同一 ID 対で判定し直して確定
4. **explained_ratio**: 8フィードで退行なし (八戸 0.9993 / 地鉄・徳島・臨港・
   奥州 1.0000 / 豊前 0.9922→同等 / 芦屋 0.9859→同等)。
   永井のみ 0.9997 — これは新世代 (2026-04-01) が公開されて比較ペアが変わり、
   「trips.txt の bikes_allowed 列削除」という**列レベル差分を説明するルールが
   元々無い**ため (M8 と無関係の既存ギャップ。残差カタログ行き)
5. **性能**: docs/perf/M8_matching_timings.md — 最大級フィードで数秒台

## 残課題

- 列レベル差分 (column_added/removed) の説明ルールが無い (永井の新世代で顕在化)
- family 対応なし・day_type 変更のブロック跨ぎ対応 (v1 制限、実害は軽微)


## 追記 (2026-07-09): ブロッキングを family → route_group に拡大

ユーザーの反例 (徳島 加茂谷線 平日): 旧 7:09 (202 加茂谷線)×新 7:20
(204 加茂谷線) など5対が、LCS 0.93・Δt 5〜18分・**trip_id まで同一**なのに
family 境界で候補から外れ、廃止+新設になっていた。事業者が便を枝番違いの
route へ移す運用は普遍的 (実測: 徳島 removed の 23% = 52便、永井 4便が
救済可能型。地鉄・八戸・豊前・芦屋・奥州は 0)。

修正: ブロックを (family, day_type) → **(route_group, day_type)** に拡大
(M6/M7 で確立済みの路線ブランド層を再利用 — 概念追加なし)。名称変更 family は
family_links 経由で新側グループに写す。

検証: 指摘5対すべて回収 (加茂谷 7:09→7:20 ほか)。徳島 removed 227→176。
クリーン正解 recall 0.9916 (不変)・explained_ratio 8フィード不変・
テスト159件通過。
