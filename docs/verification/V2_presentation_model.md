## 実装メモ・観察

- 方向グループの canonical (forward の向き) は 便数 → **最も早い始発** → system_id の
  タイブレーク (始発が早い側が起点、という自然な規則)
- 地鉄のように trip_id に時刻が埋め込まれるフィードでは、時刻変更が
  removed+added 対として現れ、④の差分表示では「廃/新」の列ペアになる
  (modified 対にならないため changed_positions は付かない)。データに忠実な挙動だが、
  V3 レビューで「似た列の近接配置」等の見せ方を検討する余地あり (未確定事項に追加せず、
  V3 の観察事項として本ログに記録)
- ビューモデルの JSON サイズ: 永井 553KB / 地鉄 R8 1.4MB / 八戸 2.6MB
  (バンドル追加分として許容範囲)
