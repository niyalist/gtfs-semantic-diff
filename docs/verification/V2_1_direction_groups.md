# V2.1 検証: 方向グループ規則の一般化 (端点完全一致 → 順序整合度)

実施日: 2026-07-07。経緯・判断・DoD は roadmap.md V2.1、規則は presentation.md
「表示規約 — 方向グループ」と R15 改訂注記 (凍結後の改訂履歴) を参照。

## 変更内容

- `report/presentation.py`
  - `order_agreement(stops_a, stops_b)` を新設: 共有停留所ペアのうち相対順序が
    一致する割合 (初出位置で判定 — 循環線の重複停留所対策) と共有停留所数を返す
  - `_direction_groups` の union 条件: 「Jaccard ≥ 0.6 かつ 端点完全逆転/完全一致」
    → 「Jaccard ≥ 0.6 かつ 共有停留所数 ≥ 3 かつ (整合度 ≤ 0.2 = 往復 or
    整合度 ≥ 0.8 = 同方向)」。中間域 (0.2〜0.8) は束ねない
  - leg 割当: 「代表と端点完全逆転か」→「代表との順序整合度 ≤ 0.2 で reverse」
- `config/default.toml [presentation]`: `direction_reversed_max_agreement = 0.2`,
  `direction_same_min_agreement = 0.8`, `direction_min_shared_stops = 3`
- identity 層 (系統=パターンクラスタ)・コア・説明会計・JSON スキーマは不変。
  束ね範囲も route_group ページ内のまま

## 目視観察と残論点 (V3 レビューで確認)

1. **前橋玉村線が1つの大きな方向グループ (系統14) に統合** — 区間便群が
   leg ごとの1枚の時刻表に併合される (区間外は空欄セル)。出版時刻表の慣行に
   近づく方向だが、表が大きくなるため見え方はユーザーレビューで確認する。
2. **荻窪公園線: 小坂子・嶺公園・荻窪公園の枝先違いが同一方向グループ** —
   幹線部の共有が Jaccard ≥ 0.6 を満たすため。過併合ではなく規則通りの挙動
   (枝は時刻表上 空欄/‖ セルで区別される) と判断するが、これもレビュー対象。
3. 起点の大きく異なる循環線同士 (回転が大きい場合) は中間域に落ちて併合されない
   (保守的側の挙動、実害未観測)。
